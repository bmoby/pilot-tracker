# Ручной ИИ-анализ выбранного обновления

## Цель

Как преподаватель, я хочу вручную запустить ИИ-анализ конкретного обновления с новой работой студента, чтобы быстро понять человеческим языком, что было добавлено, и сохранить финальное решение за собой.

## Связанные спецификации

- Бизнес-контекст: ИИ помогает, но не заменяет преподавателя.
- `Ф004`: ИИ-анализ запускается вручную только для выбранного обновления.
- `Ф002`: обновление сохраняет новый коммит и старый коммит, если он известен.
- `Ф003`: ИИ-рапорт отображается внутри выбранного обновления.
- `Т009`: запуск выполняется через локальный `codex exec` в режиме чтения.
- `Т010`: ошибка начатого ИИ-анализа сохраняется как неуспешный ИИ-рапорт.

## Лучший сценарий

1. Преподаватель открывает детали первой загрузки проекта или обновления с новыми коммитами.
2. Нажимает запуск ИИ-анализа.
3. Приложение проверяет доступность Codex CLI.
4. Приложение готовит review-копию или безопасную папку анализа.
5. Приложение выбирает режим анализа: текущее состояние проекта для первой загрузки или изменения между коммитами для обновления с новыми коммитами.
6. Приложение формирует входные данные: студент, проект, ветка, режим анализа, старый коммит, если он известен, новый коммит, количество коммитов, предыдущие комментарии и предыдущие ИИ-рапорты этого обновления.
7. Если доступен GitHub CLI, приложение пытается получить контекст pull request.
8. Приложение запускает `codex exec` в режиме чтения, передавая промпт через stdin.
9. Результат сохраняется как новый ИИ-рапорт внутри выбранного обновления.
10. Преподаватель читает краткое резюме, что добавлено как функциональность, что можно быстро проверить руками, вопросы студенту и черновик комментария.
11. Преподаватель сам решает, использовать ли черновик и какой статус выставить.

Ожидаемый результат: ИИ-рапорт связан с конкретным коммитом или диапазоном коммитов и не меняет статус проверки автоматически.

## Рабочие отклонения

### Контекст pull request недоступен

Приложение сохраняет диагностическую ошибку контекста pull request, но продолжает анализ по локальному Git-контексту выбранного режима, если он доступен.

### Старый коммит недоступен при первой загрузке

Приложение не считает это ошибкой. ИИ анализирует текущее состояние проекта на новом коммите: что это за проект, что уже можно увидеть или попробовать, какие основные части видны и что преподавателю стоит проверить руками.

### В обновлении нет новых коммитов

Приложение не запускает ИИ-анализ и не создает новый ИИ-рапорт. Интерфейс объясняет, что в этом обновлении нет новой работы студента для анализа, и преподаватель может посмотреть предыдущие сохраненные рапорты или выбрать обновление с изменениями.

### Преподаватель запускает анализ повторно

Приложение создает новый ИИ-рапорт и не перезаписывает старые рапорты. Предыдущие рапорты могут быть переданы в контекст, если это нужно для повторного анализа выбранного обновления.

### Codex CLI недоступен

Если приложение заранее знает из настроек или диагностики, что Codex CLI недоступен, действие запуска отключено и новый ИИ-рапорт не создается. Интерфейс объясняет, что нужно исправить Codex CLI в настройках.

Если преподаватель нажал запуск, приложение приняло попытку анализа, но уже во время запуска Codex CLI оказался недоступен, не авторизован или завершился ошибкой, приложение сохраняет неуспешный ИИ-рапорт со статусом ошибки внутри выбранного обновления. Страница студента не ломается, статус проверки не меняется.

## Худший допустимый сценарий

У обновления нет надежного состояния для анализа: отсутствует локальный путь, новый коммит или новая работа студента.

Приложение не запускает ИИ. Действие недоступно или завершается понятной ошибкой, которая сохраняется как часть состояния выбранного обновления.

## Шаги и опора на спецификации

| Шаг | Действие | Почему это разрешено спецификациями |
| --- | --- | --- |
| 1 | Запускать ИИ только вручную | Бизнес-контекст, `Ф004` и `Т009` запрещают автоматический ИИ-запуск. |
| 2 | Запускать из выбранного обновления | `Ф003` и `Ф004` связывают ИИ-анализ с деталями обновления. |
| 3 | Использовать данные обновления из `Ф002` | `Ф002` сохраняет новый коммит, ветку, локальный путь, старый коммит и количество новых коммитов, если оно известно. |
| 4 | Запускать локальный Codex CLI | `Ф004`, `Ф005` и `Т009` исключают прямой OpenAI API и альтернативных провайдеров. |
| 5 | Работать в read-only | `Ф004` и `Т009` запрещают изменять файлы, commit и push. |
| 6 | Сохранять несколько рапортов | `Ф003` и `Ф004` разрешают несколько ИИ-рапортов у одного обновления. |
| 7 | Не менять статус автоматически | Бизнес-контекст и `Ф004` оставляют решение преподавателю. |
| 8 | Не запускать ИИ без новых коммитов | `Ф003`, `Ф004` и `Т009` ограничивают запуск первой загрузкой проекта или обновлением с новой работой студента. |

## Границы

Эта история не добавляет автоматическую оценку, публикацию комментария в GitHub, ручной выбор режима рапорта, альтернативные ИИ-провайдеры, хранение OpenAI API-ключей или прямой вызов OpenAI API из приложения.
