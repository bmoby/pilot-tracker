# Первое открытие приложения и подготовка локальной среды

## Цель

Как преподаватель, я хочу впервые открыть Pilot Tracker и понять, готово ли мое локальное окружение к работе, чтобы не начинать проверку студентов с неясного состояния.

## Связанные спецификации

- Бизнес-контекст: приложение работает локально для одного преподавателя и одной активной когорты.
- `Ф005`: все данные хранятся локально в `data/`, настройки включают Git, GitHub CLI, Codex CLI и VS Code.
- `Т004`: приложение создает папки и JSON-файлы безопасно.
- `Т007`: первый рабочий экран - список студентов, а настройки доступны из основной навигации.
- `Т008`: настройки показывают пути и диагностику локальных инструментов.
- `Т010`: ошибки окружения должны быть понятными и не должны скрывать техническую диагностику.

## Лучший сценарий

1. Преподаватель запускает приложение.
2. Приложение создает структуру `data/`, если ее еще нет.
3. Приложение создает пустые JSON-файлы в `data/app/`.
4. Приложение открывает главный экран списка студентов.
5. Преподаватель открывает настройки.
6. Приложение показывает, что Git, GitHub CLI, Codex CLI и VS Code доступны.
7. Преподаватель возвращается к списку студентов и готов начать работу.

Ожидаемый результат: локальная среда готова, данные будут сохраняться в папке приложения, а преподаватель видит пустой рабочий список без маркетингового экрана, авторизации или выбора курса.

## Рабочие отклонения

### GitHub CLI недоступен

Приложение показывает, что GitHub CLI недоступен. Это не блокирует добавление студентов, локальное хранение данных или обновление публичных репозиториев через Git, но контекст pull request для ИИ-анализа позже будет недоступен.

### Codex CLI недоступен

Приложение показывает, что Codex CLI недоступен. Это не блокирует каталог студентов, обновление проектов, комментарии и статусы, но ручной ИИ-анализ будет недоступен до исправления настройки.

### VS Code недоступен

Приложение показывает, что команда открытия VS Code недоступна. Это не блокирует обновления, комментарии и ИИ-рапорты, но действие открытия кода в VS Code будет недоступно или завершится понятной ошибкой.

## Худший допустимый сценарий

Папку `data/` или JSON-файлы нельзя создать из-за прав доступа или ошибки диска.

В этом случае приложение не должно делать вид, что работа началась успешно. Оно показывает критичную ошибку хранения, путь, с которым возникла проблема, и не записывает частичные данные поверх поврежденного состояния.
