# Первое открытие приложения и подготовка локальной среды

## Цель

Как преподаватель-администратор, я хочу впервые открыть Pilot Tracker, войти при необходимости и понять, готово ли мое локальное окружение к работе, чтобы не начинать проверку студентов с неясного состояния.

## Связанные спецификации

- Бизнес-контекст: приложение работает для одного преподавателя и одной активной когорты с защищенным администраторским доступом.
- `Ф005`: прикладные данные хранятся в Supabase, а локальными остаются настройки, репозитории, review-копии, Git, GitHub CLI, Codex CLI и VS Code.
- `Т004`: приложение создает локальные папки и файлы безопасно.
- `Т007`: без сессии показывается вход администратора, после входа первый рабочий экран - список студентов, а настройки доступны из основной навигации.
- `Т008`: настройки показывают пути и диагностику локальных инструментов.
- `Т010`: ошибки окружения должны быть понятными и не должны скрывать техническую диагностику.
- `Т011`: Supabase является источником истины для прикладных данных.
- `Т012`: Supabase Auth защищает рабочие данные администраторской сессией.

## Лучший сценарий

1. Преподаватель запускает приложение.
2. Если сессии нет, приложение показывает вход администратора.
3. Преподаватель входит по email и паролю администратора.
4. Приложение открывает главный экран списка студентов из Supabase.
5. Приложение создает локальную структуру `data/`, если она нужна для настроек, журналов и локальных артефактов.
6. Преподаватель открывает настройки.
7. Приложение показывает, что Git, GitHub CLI, Codex CLI и VS Code доступны.
8. Преподаватель возвращается к списку студентов и готов начать работу.

Ожидаемый результат: администратор вошел, прикладные данные доступны из Supabase, локальная среда готова, а преподаватель видит рабочий список без маркетингового экрана или выбора курса.

## Рабочие отклонения

### Сессии нет

Приложение показывает вход администратора и не загружает рабочие данные до успешной авторизации.

### Supabase недоступен

После входа приложение показывает ошибку подключения к Supabase. Создание и изменение прикладных данных блокируются, чтобы не появлялся второй источник истины.

### GitHub CLI недоступен

Приложение показывает, что GitHub CLI недоступен. Это не блокирует просмотр данных Supabase и комментарии, но контекст pull request для ИИ-анализа позже будет недоступен.

### Codex CLI недоступен

Приложение показывает, что Codex CLI недоступен. Это не блокирует каталог студентов, обновление проектов, комментарии и статусы, но ручной ИИ-анализ будет недоступен до исправления настройки. Новые ИИ-рапорты из-за такого заранее известного состояния не создаются.

### VS Code недоступен

Приложение показывает, что команда открытия VS Code недоступна. Это не блокирует обновления, комментарии и ИИ-рапорты, но действие открытия кода в VS Code будет недоступно или завершится понятной ошибкой.

## Худший допустимый сценарий

Папку `data/`, локальные настройки или диагностические журналы нельзя создать из-за прав доступа или ошибки диска.

В этом случае приложение не должно делать вид, что локальная среда готова. Оно показывает критичную ошибку локального хранения, путь, с которым возникла проблема, и не записывает частичные данные поверх поврежденного состояния.
