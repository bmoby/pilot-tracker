# Дорожная карта

Эта папка хранит хронологическую историю выполненных работ по Pilot Tracker.

Дорожная карта обновляется каждый раз, когда разработка по плану завершена, владелец проекта проверил результат и попросил подготовить изменения к отправке в GitHub.

Перед commit, push и созданием pull request агент должен обновить:

- `хронология.md` - общий список завершенных работ по датам;
- один или несколько файлов в `области/` - подробности по затронутым областям проекта.

Базовые области дорожной карты:

- `области/студенты-и-проекты.md` - каталог студентов, карточки, связь студента с проектом и текущие статусы;
- `области/github-и-репозитории.md` - GitHub, локальные репозитории студентов, обновления, review-копии и открытие кода;
- `области/проверка-и-история.md` - лента обновлений, комментарии, статусы проверки и история ревью;
- `области/данные.md` - модель данных, локальное хранение, JSON-схемы, настройки путей и переносимость данных;
- `области/фронтенд-и-дизайн.md` - страницы, навигация, состояния интерфейса, визуальная структура и пользовательский опыт;
- `области/ии-анализ.md` - локальный ИИ-анализ, ИИ-рапорты, черновики комментариев и проверяемость результата ИИ;
- `области/инфраструктура-и-качество.md` - архитектура, структура проекта, инструменты, диагностика и тестирование.

Если завершенный план затрагивает новую область, для нее нужно создать отдельный файл в `области/` и добавить запись в хронологическом порядке.

Записи в дорожной карте должны быть краткими, проверяемыми и связанными с конкретным планом разработки.

Новые записи добавляются в конец соответствующих файлов, чтобы порядок оставался хронологическим от старых работ к новым.
