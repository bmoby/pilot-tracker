## Цель

Добавить рабочую диагностику локальной среды на странице настроек, чтобы преподаватель видел доступность путей данных и локальных инструментов до запуска операций, которые зависят от Git, GitHub CLI, Codex CLI и VS Code.

## Контекст спецификаций

Прочитаны бизнес-контекст, `Ф005`, `Ф004`, `Т008`, `Т009` и `Т010`. План остается внутри рамок первой локальной версии: приложение проверяет только локальные пути и команды, не хранит секреты, не управляет авторизацией GitHub или Codex и не запускает ИИ-анализ.

## Входит в план

- Добавить действие `Проверить окружение` на странице настроек.
- Проверять доступность папок `data/`, `data/app/`, `data/repositories/`, `data/review-copies/` и возможность записи настроек.
- Проверять команды `git --version`, `gh --version`, `gh auth status`, `codex --version` и `code --version`.
- Сохранять результат последней диагностики в `data/app/settings.json`.
- Показывать статусы, версии, сообщения и время последней проверки на странице настроек.
- Покрыть диагностику тестами через управляемые заглушки процессов и временное хранилище.

## Не входит в план

- Ручной запуск ИИ-анализа выбранного обновления.
- Создание и отображение ИИ-рапортов.
- Настройка собственных токенов GitHub или OpenAI внутри приложения.
- Редактирование команд инструментов через интерфейс.
- Автоматическое исправление локального окружения преподавателя.

## Шаги разработки

1. Уточнена текущая модель настроек и хранилища для сохранения результатов диагностики.

6. Выполнена автоматическая и ручная проверка страницы настроек.

## Критерии готовности

- На странице настроек есть действие `Проверить окружение`.
- После проверки статусы сохраняются и остаются видимыми после перезагрузки страницы.
- Недоступность GitHub CLI отображается как ограничение контекста pull request, а не как полная поломка приложения.
- Недоступность Codex CLI явно показывает, что ИИ-анализ нельзя будет запустить до исправления.
- Недоступность VS Code явно показывает, что открытие кода ограничено.
- Диагностика не меняет студентов, проекты, обновления, комментарии, статусы проверки и ИИ-рапорты.
- Тесты и проверка качества проходят.

## Пользовательская проверка

Владелец проекта открывает `http://localhost:3000/settings`, нажимает `Проверить окружение`, проверяет, что Git, GitHub CLI, Codex CLI и VS Code показывают понятные статусы, версии или причины недоступности, затем перезагружает страницу и убеждается, что результат последней проверки сохранился.

## Риски и открытые вопросы

- `gh auth status` может вернуть предупреждение, если GitHub CLI установлен, но пользователь не авторизован; это не должно блокировать базовую работу приложения.
- `codex --version` проверяет только наличие Codex CLI, но не гарантирует успешный будущий запуск анализа проекта.
- Команда `code --version` может быть недоступна на macOS, даже если приложение Visual Studio Code установлено; для первой диагностики это показывается как ограничение команды `code`.
