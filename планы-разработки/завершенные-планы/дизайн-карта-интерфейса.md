# Дизайн-карта интерфейса

- Статус: завершен
- Ветка: `plan/2026-05-14-design-map`
- Дата создания: 2026-05-14
- Связанные спецификации: `спецификации/бизнес-контекст.md`, `спецификации/технические/Т002-структура-проекта-и-runtime/спецификация.md`, `спецификации/технические/Т007-frontend-навигация-и-страницы/спецификация.md`
- Области дорожной карты: `дорожная-карта/области/фронтенд-и-дизайн.md`, `дорожная-карта/области/инфраструктура-и-качество.md`

## Цель

Создать отдельную страницу дизайн-карты Pilot Tracker, которая использует предоставленные визуальные референсы только как направление стиля и фиксирует набор интерфейсных элементов, реально нужных рабочим экранам первой версии продукта.

## Контекст спецификаций

Бизнес-контекст фиксирует Pilot Tracker как локальное рабочее место преподавателя, а не маркетинговый сайт или студенческий кабинет. `Т002` разрешает реализовывать интерфейс на Next.js и хранить интерфейсные компоненты в `src/ui/`, при этом не смешивать их со сценариями приложения. `Т007` описывает рабочие экраны первой версии и запрещает расширять продуктовую навигацию лишними разделами. Поэтому дизайн-карта должна быть внутренней справочной страницей для разработки, доступной по прямому маршруту, но не частью основной навигации преподавателя.

## Входит в план

- Уточнить `Т007`, что внутренняя страница дизайн-карты может существовать как справочник для будущего редизайна.
- Создать отдельный маршрут `/design-map`.
- Разобрать спецификации и существующие рабочие экраны, чтобы не добавлять в дизайн-карту лишние продуктовые элементы.
- Зафиксировать UI-инвентарь, который нужен для покрытия будущего интерфейса Pilot Tracker: состояния студентов, строки списка, события обновлений, рабочую область проверки, ИИ-рапорты, комментарии, формы, уведомления и диагностику.
- Показать маленькие компоненты в стиле референсов, но с демонстрационными данными и состояниями Pilot Tracker.
- Сделать страницу статической: без чтения данных, без Git, Codex, GitHub CLI, VS Code и локального хранилища.
- Проверить страницу в браузере на широком и узком экране.

## Не входит в план

- Переделка рабочих экранов `Студенты`, страницы студента или `Настройки`.
- Добавление дизайн-карты в основную навигацию преподавателя.
- Изменение данных, сценариев приложения, хранилища или интеграций.
- Копирование бренда, названия или пользовательских текстов из референса.
- Реализация интерактивной логики компонентов.
- Создание полноценной дизайн-системы компонентов для повторного импорта в продуктовые экраны.

## Шаги разработки

1. Обновить `Т007`, чтобы зафиксировать внутреннюю страницу дизайн-карты.
2. Создать маршрут `/design-map` и компонент страницы дизайн-карты.
3. Сверстать дизайн-карту на основе спецификаций и UI-инвентарь маленьких компонентов на русском языке.
4. Проверить адаптивность страницы и отсутствие конфликтов с рабочими экранами.
5. Выполнить автоматические проверки проекта.

## Критерии готовности

- Страница `/design-map` открывается в локальном интерфейсе.
- На странице есть список UI-компонентов, необходимых для покрытия будущих экранов Pilot Tracker.
- На странице показаны маленькие визуальные элементы: цвета, заголовки, подзаголовки, кнопки, поля, строки списка, карточки обновлений, статусы, уведомления и диагностика.
- На странице нет продуктовых элементов, которые не описаны спецификациями первой версии, например прогресса студента, бизнес-листингов или сторонних рабочих сущностей.
- На странице есть графическая карта для будущего редизайна Pilot Tracker без изменения рабочих экранов.
- Все тексты на странице написаны на русском языке.
- Страница не добавлена в основную навигацию приложения.
- Рабочие экраны продукта не изменены.
- `pnpm check` проходит успешно.

## Пользовательская проверка

Владелец проекта открывает `/design-map`, проверяет полноту разбора референса, визуальную близость принципов к желаемому направлению и подтверждает, что страница подходит как эталон для будущего редизайна.

## Риски и открытые вопросы

- Нужно взять из референса визуальные принципы, но не копировать чужой бренд и продуктовые тексты.
- Дизайн-карта не должна случайно стать обещанием новых продуктовых функций.
- После приемки дизайн-карты отдельным планом потребуется применить выбранные решения к рабочим экранам Pilot Tracker.
